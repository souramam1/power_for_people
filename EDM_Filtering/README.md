# Power For People — EDM audit scripts

This folder contains the Python scripts and data behind the
"has this MP signed an EDM?" audit used to populate the MPs & Peers
Database.

The main script is **`src/EDM_Signatory_General_Check.py`**. It talks to
the UK Parliament's open APIs, downloads every Early Day Motion tabled
since a chosen start date, and works out which MPs (or Peers) signed
which motions.

**Note on the use of AI:** This codebase was generated using Claude Opus 4.7 (LLM), over a time period of ~3hrs. The number of tokens used during this process is unknown. Subsequent use of the code within the codebase will *not* call on any MCP (model context protocols), LLMs or use any API's apart from parliamentary APIs specified in the documentation below.

## Three-step setup (do these once, in order)

> **Important:** `requirements.txt` only lists **Python libraries**
> (`requests`, `openpyxl`). It does **not** install Python itself.
> You must install Python first (Step 1) before `pip` in Step 2 will
> work.

### Step 1 — Install Python

Check whether Python is already installed. Open a terminal (Start menu →
type "Terminal" or "PowerShell" → Enter) and run:

```
python --version
```

- If you see `Python 3.10.x` or higher, skip to Step 2.
- If you see `'python' is not recognized` or a version below 3.10,
  download Python from <https://www.python.org/downloads/>.
  **Tick the box labelled "Add Python to PATH"** in the installer, then
  finish the installer and **open a new terminal** before running
  `python --version` again.

### Step 2 — Install the required Python libraries

Open a terminal in the **EDM_Filtering** folder (the folder that
contains this README). On Windows, the easiest way is:

1. Open File Explorer and navigate to
   `C:\Users\DELL\OneDrive\Power_For_People\Code\EDM_Filtering`.
2. Hold **Shift**, right-click in empty space in the folder, and pick
   **"Open in Terminal"** (or "Open PowerShell window here").

Then run:

```
python -m pip install -r requirements.txt
```

This reads `requirements.txt` and installs the two libraries the scripts
need (`requests` and `openpyxl`). You only do this once per computer.

### Step 3 — Run the main script

From the same terminal in the EDM_Filtering folder:

```
python src\EDM_Signatory_General_Check.py
```

- **First run:** 30–60 minutes — it fetches every EDM one at a time.
- **Later runs:** 1–5 minutes — only NEW EDMs since your last run are
  fetched. Status CSVs that only change filters are rebuilt in seconds.

The main script produces two files inside `databases\edm_data\`:

| File | What's in it |
|---|---|
| `databases\edm_data\edm_sponsors.csv` | **Master data.** One row per EDM, one copy of this file total — it's the same for every run, so the script updates it in place rather than duplicating it. |
| `databases\edm_data\edm_signing_status.csv` | **Default run output** (when no filters are active). Stable filename so other tools can point at it. |
| `databases\edm_data\<label>_<YYYY-MM-DD_HH-MM-SS>\edm_signing_status.csv` | **Filtered run output.** Each filtered run gets its own subfolder named after the filter label + the date/time it was run. Inside, the signing table is called simply `edm_signing_status.csv`. |

A matching `.xlsx` with both tables as sheets is written alongside each
CSV (unless you set `WRITE_XLSX = False`).

## About the cache — and what happens when you share this folder

The main script creates a folder called **`cache\edm\`** at the top of
this project. Inside it are ~3,000 tiny JSON files — one per EDM —
copied straight from the Parliament API. The only reason it exists is
to make re-runs of the script fast: if a cached file is present, the
script skips the network call for that EDM.

**You should not share the `cache\` folder.**

- It's large (tens of MB).
- It's not data — it's a speed-up. Anyone running the script regenerates
  it automatically on their first run.
- `.gitignore` already excludes it from git, so if you share via git
  (GitHub, etc.) it will NOT be included.
- If you share by **zipping the EDM_Filtering folder** and sending it,
  **delete the `cache\` folder first** (or configure your archiver to
  exclude it).

**When to delete the cache yourself:** if you ever suspect the cached
data is stale (e.g. an EDM has new signatories but your output doesn't
reflect them), delete the matching `cache\edm\<edm_id>.json` file — or
the whole `cache\` folder — and re-run.

The master `edm_sponsors.csv` (in `databases\edm_data\`) is a stable
record of what was true at each run. If you delete that, the next run
rebuilds it from scratch (slow — because the cache is what keeps it
fast, and with no cache the script has to re-fetch every EDM).

## Configuring the main script

Open `src\EDM_Signatory_General_Check.py` in any text editor (Notepad,
VS Code, whatever) and find the `CONFIG` block near the top. Edit the
values inside it, save, and re-run the script.

Everything outside the CONFIG block is internal logic — you don't need
to touch it.

### The seven knobs

| Setting | What it does |
|---|---|
| `START_DATE`, `END_DATE` | The date window for EDMs. `END_DATE = None` means "up to today". |
| `HOUSE` | `"commons"` (default) or `"lords"`. Peers don't sign EDMs, so the Lords mode is mostly useful if you just want a current Peers list. |
| `PRAYER_MODE` | `"all"`, `"exclude_prayers"`, or `"only_prayers"`. Prayers are procedural EDMs against statutory instruments; usually you want to exclude them. |
| `PARTY_FILTER` | `None`, or a list like `["Labour"]`. Only MPs from these parties appear in the output. How the list is matched is controlled by `PARTY_FILTER_MODE`. |
| `PARTY_FILTER_MODE` | `"substring"` (default) or `"exact"`. Substring means `["Labour"]` matches both `Labour` and `Labour (Co-op)`, and `["Green"]` matches `Green Party`. Exact means the MP's party string must equal a list entry verbatim (case-insensitive). Use `"exact"` when you need to exclude Labour (Co-op) from a pure-Labour run. |
| `CONSTITUENCY_FILTER` | `None`, or a list like `["Cambridge", "Brighton Pavilion"]`. Only MPs from these seats appear. |
| `KEYWORD_FILTER` | `None`, or a list like `["climate", "net zero"]`. An EDM counts only if its title contains at least one of these words (case-insensitive). |
| `RUN_LABEL` | `None` (auto-derive a label from the filters above) or a custom string like `"climate_push_2026"`. Goes into the output filename. |

### Three worked examples

**Example A: "Which Labour MPs have signed a climate EDM?"**

```python
PARTY_FILTER   = ["Labour"]
KEYWORD_FILTER = ["climate", "net zero", "renewable", "clean power"]
RUN_LABEL      = "labour_climate"
```

Output: a new subfolder
`databases\edm_data\labour_climate_2026-04-24_15-32-08\` containing
`edm_signing_status.csv` (one row per Labour MP with yes/no on whether
they signed any climate-themed EDM) and a matching `.xlsx`. The folder
name's date/time is from when you ran the script, so re-running never
clobbers an earlier result.

**Example B: "Regular (non-prayer) EDM signing rates across all MPs"**

```python
PRAYER_MODE = "exclude_prayers"
```

Output: subfolder
`databases\edm_data\exclude_prayers_2026-04-24_15-32-08\` with the
signing CSV inside. All 650 MPs, counts exclude prayer EDMs.

**Example C: "Just the Lib Dems and the Greens on health-related motions"**

```python
PARTY_FILTER   = ["Liberal Democrat", "Green Party"]
KEYWORD_FILTER = ["NHS", "health", "hospital", "GP"]
RUN_LABEL      = "libdem_green_health"
```

Output: subfolder
`databases\edm_data\libdem_green_health_2026-04-24_15-32-08\`.

### Tips

- `PARTY_FILTER` defaults to **substring matching**, so shorthand like
  `"Green"` matches `"Green Party"` and `"Labour"` matches both
  `"Labour"` and `"Labour (Co-op)"`. To disable this and require exact
  matches, set `PARTY_FILTER_MODE = "exact"`.
- If a filtered run comes back empty, check the `party` column in
  `edm_sponsors.csv` for the exact name Parliament uses — the script
  will also print the distinct parties it actually saw.
- Keyword matching is a plain substring check — `"climate"` will match
  `"climate change"`, `"climate crisis"`, `"climate-related"`, etc.
- To reach *further back* than the current master covers, delete
  `databases\edm_data\edm_sponsors.csv` first, change `START_DATE`, and
  re-run. The script will rebuild the master from the new start date
  (slow — ~1 hour).

## Step 4 (optional) — Update the MPs & Peers database with a yes/no column

Use `src\update_mp_database.py` to take any Excel workbook with a list
of MPs and add a new column showing whether each MP signed any EDM
matching the filters you most recently ran.

### How to use it

1. Open `src\update_mp_database.py` in a text editor.
2. In the `CONFIG` block at the top, edit these two values:

   ```python
   SOURCE_XLSX       = r"databases/original_db/MPs and Peers Database.xlsx"
   NEW_COLUMN_HEADER = "Signed any EDM July 2024 onward [excl. Prayers]"
   ```

   - **`SOURCE_XLSX`**: paste the path to your Excel file. Relative
     paths are resolved against the EDM_Filtering folder, so
     `"databases/original_db/..."` works directly. You can also paste an
     absolute path — the easiest way on Windows is to right-click the
     file in File Explorer, pick **"Copy as path"**, and paste between
     the quotes (the `r"..."` prefix lets you keep backslashes as-is).
   - **`NEW_COLUMN_HEADER`**: whatever you want the new column to be
     called. If the workbook already has a column with this exact name,
     the script overwrites its values (useful for re-runs).

3. Optionally, change these:

   ```python
   STATUS_CSV          = "databases/edm_data/edm_signing_status.csv"
   STATUS_VALUE_COLUMN = "signed_edm_non_prayer_since_2024_07"
   ```

   If you just ran a **filtered** version of the main script, point at
   that run's subfolder — filtered outputs live inside a folder named
   after the filter label and run date/time, e.g.
   `"databases/edm_data/labour_climate_2026-04-24_15-32-08/edm_signing_status.csv"`
   — and set `STATUS_VALUE_COLUMN = "signed_matching_edm"` (that's the
   column filtered runs use).

4. Save the file, then run from the EDM_Filtering folder:

   ```
   python src\update_mp_database.py
   ```

The script **never modifies the source** — it writes to a new
auto-dated copy inside `databases\updated_db\` (e.g.
`MPs and Peers Database_UPDATED_24_04.xlsx`). Set `OUTPUT_XLSX` in the
CONFIG block if you want a different output location.

### What the script is safe against

The script checks everything before touching the spreadsheet and fails
with a clear message (no Python error traceback) if anything is off:

- Source file not found or not a real `.xlsx`
- You pasted a OneDrive web URL instead of a local path
- The sheet or name column can't be found (lists what WAS found)
- Status CSV missing, or the column you specified isn't in it
- `NEW_COLUMN_HEADER` is empty
- The output file is currently open in Excel

### How MP names are matched

MP names in the spreadsheet rarely match the API's formatting exactly
(middle names, diminutives, honorifics, diacritics). The script applies
three matching tiers in order:

1. **Exact** — normalised "First Last" match (case/diacritic-insensitive).
2. **Token subset** — handles middle names (`Mary Foy` ↔ `Mary Kelly Foy`).
3. **First initial + surname overlap** — handles diminutives
   (`Peter` ↔ `Pete`, `Edward` ↔ `Ed`).

Cells filled via tiers 2 or 3 are **highlighted yellow** in the output
so you can eyeball-verify them. Unmatched rows are left blank and
listed on screen so you can fix them by hand.

### Sheet / column auto-detection

The script auto-finds the MP sheet by name (tries `MPs`, `Members`,
`Commons`, etc.) and the name column (tries `First Name` + `Last Name`
first, then `Name`, `MP Name`, `Full Name`, etc.). If your workbook
uses something unusual, set these explicitly in the CONFIG block:

```python
SHEET_NAME         = "Commons MPs"
FIRST_NAME_COLUMN  = "Given"
LAST_NAME_COLUMN   = "Family"
# or, for a single-cell full name:
FULL_NAME_COLUMN   = "Representative"
```

## Output file reference

### `databases\edm_data\edm_sponsors.csv` (the master)

| Column | What it is |
|---|---|
| `edm_id` | Parliament's internal ID for the EDM |
| `date_tabled` | Date the EDM was tabled, `YYYY-MM-DD` |
| `is_prayer` | `yes` if this prays against a statutory instrument, else `no` |
| `title` | The EDM's title |
| `tabler_id`, `tabler_name` | The primary sponsor (the MP who tabled it) |
| `sponsor_count` | How many active (non-withdrawn) signatures |
| `sponsor_ids` | Semicolon-delimited list of member IDs who signed |
| `sponsor_names` | Semicolon-delimited list of signatory names |

### `databases\edm_data\edm_signing_status.csv` (default, no filters)

| Column | What it is |
|---|---|
| `member_id`, `name`, `constituency`, `party` | The MP |
| `signed_edm_since_2024_07` | `yes/no` — did they sign any EDM in the window? |
| `edms_signed_count` | How many EDMs they signed |
| `signed_edm_non_prayer_since_2024_07` | `yes/no` — any non-prayer EDMs? |
| `edms_signed_non_prayer_count` | How many non-prayer EDMs they signed |

### `databases\edm_data\<label>_<YYYY-MM-DD_HH-MM-SS>\edm_signing_status.csv` (filtered runs)

| Column | What it is |
|---|---|
| `member_id`, `name`, `constituency`, `party` | The MP |
| `signed_matching_edm` | `yes/no` — did they sign any EDM matching your filters? |
| `matching_edms_signed_count` | How many matching EDMs they signed |

## Troubleshooting

**`ModuleNotFoundError: No module named 'requests'`** (or `openpyxl`)
→ You haven't installed the libraries. From the EDM_Filtering folder,
run: `python -m pip install -r requirements.txt`

**`python: command not found` / `'python' is not recognized`**
→ Python isn't on your PATH. Go back to Step 1: reinstall Python and
tick the "Add Python to PATH" box in the installer. After installing,
close and reopen the terminal.

**`can't open file '.../EDM_Signatory_General_Check.py'`**
→ You're running the script from the wrong folder. Make sure your
terminal is in the EDM_Filtering folder (the one containing `src\`,
`databases\`, and this README), then run
`python src\EDM_Signatory_General_Check.py`.

**The main script stalls for a long time at "fetched N/M EDMs"**
→ Parliament's API occasionally rate-limits. The script backs off and
retries automatically (wait up to 2 minutes). If it fails entirely,
re-run — progress is saved in `cache\edm\` so you won't lose earlier
work.

**SSL errors on a corporate / VPN network**
→ Some corporate networks intercept HTTPS and break Python's SSL. Try
running the script on a personal network, or ask IT for the company's
CA certificate bundle and point Python at it via `SSL_CERT_FILE=<path>`.

**My filter came back empty**
→ Check the `party` or `constituency` column in `edm_sponsors.csv` for
Parliament's exact spelling. `"Lib Dem"` won't match —
`"Liberal Democrat"` will.

**I want to reach further back than 2024-07-01**
→ Delete `databases\edm_data\edm_sponsors.csv`, change `START_DATE` in
the CONFIG block of the main script, and re-run. The master will
rebuild from the new start date.

**`update_mp_database.py` says "Couldn't save ... is the file open in Excel?"**
→ Close the output workbook in Excel and run the script again. Excel
locks the file while it's open.

**`update_mp_database.py` says "Source workbook not found"**
→ Double-check `SOURCE_XLSX` in the CONFIG block. Use right-click →
"Copy as path" in File Explorer for an unambiguous path. OneDrive web
URLs (starting with `https://`) don't work — the file must be synced
to your PC.

**I see a `__pycache__\` folder I didn't create**
→ That's Python's own bytecode cache. Auto-generated, auto-ignored by
`.gitignore`, safe to delete at any time.
